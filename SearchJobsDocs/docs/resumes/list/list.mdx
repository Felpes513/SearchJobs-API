## Descricao

Retorna de forma paginada os curriculos enviados pelo usuario autenticado, ordenados do mais recente para o mais antigo. O usuario e identificado automaticamente pelo JWT informado no header.

## Autenticacao

- Necessita autenticacao: Sim
- Tipo: Bearer JWT
- Header: `Authorization: Bearer <access_token>`

## Request

### Metodo e URL

`GET /api/resumes`

### Headers

| Header        | Valor                   |
|---------------|-------------------------|
| Authorization | `Bearer <access_token>` |

### Query Params

| Parametro | Tipo    | Obrigatorio | Padrao | Descricao                          |
|-----------|---------|-------------|--------|------------------------------------|
| page      | Integer | Nao         | `0`    | Numero da pagina (comeca em 0)     |
| size      | Integer | Nao         | `10`   | Quantidade de itens por pagina     |

### Path Params

Nenhum.

### Body

Nenhum.

## Response

### Sucesso

Status: `200 OK`

```json
{
  "content": [
    {
      "id": 2,
      "fileName": "curriculo-atualizado.pdf",
      "filePath": "uploads/resumes/b2c3d4e5-curriculo-atualizado.pdf",
      "createdAt": "2024-01-20T14:00:00"
    },
    {
      "id": 1,
      "fileName": "curriculo-felipe.pdf",
      "filePath": "uploads/resumes/a1b2c3d4-curriculo-felipe.pdf",
      "createdAt": "2024-01-15T10:30:00"
    }
  ],
  "totalElements": 2,
  "totalPages": 1,
  "number": 0,
  "size": 10,
  "first": true,
  "last": true,
  "empty": false
}
```

Quando o usuario nao possui curriculos, retorna pagina vazia:

```json
{
  "content": [],
  "totalElements": 0,
  "totalPages": 0,
  "number": 0,
  "size": 10,
  "first": true,
  "last": true,
  "empty": true
}
```

### Erros

`401 Unauthorized`

Retornado quando o token JWT esta ausente ou invalido.

```json
{
  "status": 401,
  "erro": "Unauthorized",
  "mensagem": "Token ausente ou invalido",
  "path": "/api/resumes",
  "timestamp": "2024-01-15T10:30:00"
}
```

## Regras de Negocio

- Retorna somente os curriculos do usuario autenticado, nunca de outros usuarios.
- O ID do usuario e extraido diretamente do JWT, sem necessidade de parametro adicional.
- Os resultados sao ordenados por `createdAt` de forma decrescente (mais recente primeiro).

## Fluxo Resumido

1. Extrai o `userId` do JWT no header `Authorization`.
2. Busca os curriculos do `userId` com paginacao e ordenacao por data decrescente.
3. Retorna a pagina com `200 OK`.
