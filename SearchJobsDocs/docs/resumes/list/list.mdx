## Descricao

Retorna de forma paginada os curriculos enviados pelo usuario autenticado, ordenados do mais recente para o mais antigo. Quando o curriculo ja foi extraido via IA, os dados estruturados do candidato (nome, skills, experiencias, etc.) sao retornados junto. O usuario e identificado automaticamente pelo JWT informado no header.

## Autenticacao

- Necessita autenticacao: Sim
- Tipo: Bearer JWT
- Header: `Authorization: Bearer <access_token>`

## Request

### Metodo e URL

`GET /api/resumes`

### Headers

| Header        | Valor                   |
|---------------|-------------------------|
| Authorization | `Bearer <access_token>` |

### Query Params

| Parametro | Tipo    | Obrigatorio | Padrao | Descricao                          |
|-----------|---------|-------------|--------|------------------------------------|
| page      | Integer | Nao         | `0`    | Numero da pagina (comeca em 0)     |
| size      | Integer | Nao         | `10`   | Quantidade de itens por pagina     |

### Path Params

Nenhum.

### Body

Nenhum.

### Erros

`401 Unauthorized`

Retornado quando o token JWT esta ausente ou invalido.

```json
{
  "status": 401,
  "erro": "Unauthorized",
  "mensagem": "Token ausente ou invalido",
  "path": "/api/resumes",
  "timestamp": "2024-01-15T10:30:00"
}
```

## Regras de Negocio

- Retorna somente os curriculos do usuario autenticado, nunca de outros usuarios.
- O ID do usuario e extraido diretamente do JWT, sem necessidade de parametro adicional.
- Os resultados sao ordenados por `createdAt` de forma decrescente (mais recente primeiro).
- O campo `extraido` indica se a extracao via IA ja foi realizada. Se `false`, os demais campos de conteudo serao `null`.
- Para extrair os dados do curriculo, utilize `POST /api/resumes/{id}/extract`.

## Fluxo Resumido

1. Extrai o `userId` do JWT no header `Authorization`.
2. Busca os curriculos do `userId` com paginacao e ordenacao por data decrescente.
3. Para cada curriculo, verifica se `parsedJson` esta preenchido e desserializa os dados estruturados.
4. Retorna a pagina com `200 OK`.
