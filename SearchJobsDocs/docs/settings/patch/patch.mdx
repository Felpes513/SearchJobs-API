---
id: settings-patch
title: Atualizar Configurações Parcialmente
---

## Descricao

Atualiza parcialmente as configuracoes do usuario autenticado. Diferente do `PUT`, campos omitidos ou `null` sao ignorados e mantem o valor atual no banco — inclusive para `language`, `theme` e `termsAccepted`.

Use este endpoint quando quiser alterar apenas um ou dois campos sem sobrescrever os demais.

## Autenticacao

- Necessita autenticacao: Sim
- Tipo: Bearer JWT
- Header: `Authorization: Bearer <access_token>`

## Request

### Metodo e URL

`PATCH /api/settings`

### Headers

| Header        | Valor                    |
|---------------|--------------------------|
| Authorization | `Bearer <access_token>`  |
| Content-Type  | `application/json`       |

### Query Params

Nenhum.

### Body

Envie apenas os campos que deseja alterar.

```json
{
  "theme": "dark"
}
```

```json
{
  "openAiApiKey": "sk-nova-chave...",
  "jsearchApiKey": "nova-jsearch-key..."
}
```

### Campos do Body

| Campo         | Tipo    | Obrigatorio | Descricao                                                                        |
|---------------|---------|-------------|----------------------------------------------------------------------------------|
| language      | String  | Nao         | Idioma da interface (ex: `pt-BR`, `en-US`)                                       |
| theme         | String  | Nao         | Tema da interface (`light`, `dark`, `system`)                                    |
| termsAccepted | Boolean | Nao         | Aceitacao dos termos de uso                                                      |
| openAiApiKey  | String  | Nao         | Chave da API OpenAI. Omitir para manter a chave atual.                           |
| jsearchApiKey | String  | Nao         | Chave da API JSearch (RapidAPI). Omitir para manter a chave atual.              |

## Response

### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Configuracoes atualizadas com sucesso",
  "data": {
    "language": "pt-BR",
    "theme": "dark",
    "termsAccepted": true,
    "openAiApiKeyMasked": "sk-****1234",
    "jsearchApiKeyMasked": "****7890",
    "hasOpenAiApiKey": true,
    "hasJsearchApiKey": true,
    "updatedAt": "2025-04-10T15:00:00"
  }
}
```

### Erros

`401 Unauthorized`

```json
{
  "success": false,
  "message": "Token ausente ou invalido"
}
```

## Diferenca entre PUT e PATCH

| Comportamento              | PUT                          | PATCH                        |
|----------------------------|------------------------------|------------------------------|
| Campo omitido (`language`) | Definido como `null`         | Mantido o valor anterior     |
| Campo omitido (`termsAccepted`) | Definido como `false`   | Mantido o valor anterior     |
| Chave de API omitida       | Mantida (excecao das regras) | Mantida                      |

## Seguranca

- As chaves sao criptografadas com AES-256 antes de serem persistidas.
- As chaves nunca sao logadas nem retornadas em texto puro.
- Toda escrita e vinculada exclusivamente ao `userId` extraido do JWT.

## Fluxo Resumido

1. Extrai o `userId` do JWT no header `Authorization`.
2. Busca as configuracoes existentes do usuario (ou cria uma base vazia).
3. Para cada campo do body: se nao nulo, substitui o valor atual; caso contrario, mantem.
4. Criptografa as chaves e persiste no banco.
5. Retorna as configuracoes completas dentro do envelope `ApiResponse` com `200 OK`.
