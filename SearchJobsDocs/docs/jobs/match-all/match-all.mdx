## Descricao

Analisa a compatibilidade do usuario autenticado com todas as vagas disponiveis no banco de dados usando IA (OpenAI GPT). Para cada vaga, o GPT gera um score de 0 a 100, uma justificativa e listas de pontos fortes e gaps do candidato.

O fluxo funciona em tres etapas:
1. **Pre-filtragem**: seleciona ate 15 vagas com maior correspondencia de skills com o perfil do usuario.
2. **Analise por IA**: o GPT avalia a compatibilidade individualmente para cada vaga selecionada.
3. **Persistencia**: os resultados sao salvos no banco e retornados ordenados por score (maior primeiro).

:::info Pre-requisitos
Antes de executar este endpoint, o usuario precisa:
1. Configurar a **OpenAI API Key** em `PUT /api/settings`.
2. Ter vagas disponiveis no banco — utilize `GET /api/jobs/search` primeiro.

Se a chave estiver ausente, a API retorna `400` com mensagem indicando que a chave precisa ser configurada.
:::

:::tip Cache
Os resultados sao armazenados em cache por **10 minutos** por usuario. Requisicoes subsequentes dentro desse periodo retornam o resultado cacheado sem chamar o GPT novamente.

O cache e invalidado automaticamente quando o usuario extrai um novo curriculo via `POST /api/resumes/{id}/extract`, garantindo que a proxima analise reflita o perfil atualizado.
:::

:::caution Custo de processamento
Este endpoint faz uma chamada a API da OpenAI para cada vaga analisada (ate 15). O cache reduz o consumo de creditos em chamadas repetidas dentro da janela de 10 minutos.
:::

  "success": true,
  "message": "Matching realizado com sucesso",
  "data": [
    {
      "jobId": 1,
      "titulo": "Backend Developer - Java",

      "localizacao": "Sao Paulo, SP",

### Campos de `data`

| Campo          | Tipo         | Descricao                                                        |
|----------------|--------------|------------------------------------------------------------------|
| jobId          | Long         | ID interno da vaga no banco de dados                             |
| titulo         | String       | Titulo da vaga                                                   |
| empresa        | String       | Nome da empresa                                                  |
| localizacao    | String       | Localizacao da vaga                                              |
| modeloTrabalho | String       | Modelo de trabalho (Remote, Hybrid, On-site)                     |
| jobUrl         | String       | Link para a vaga original                                        |
| score          | Double       | Pontuacao de compatibilidade de 0 a 100                          |
| justificativa  | String       | Explicacao do score em 2-3 frases gerada pela IA                 |
| pontosFortres  | Array String | Skills e experiencias do candidato alinhadas com a vaga          |
| gaps           | Array String | Requisitos da vaga que o candidato nao possui                    |

### Criterios de Score

| Faixa     | Significado                                              |
|-----------|----------------------------------------------------------|
| 80 – 100  | Candidato atende a maioria dos requisitos                |
| 60 – 79   | Candidato atende requisitos basicos com alguns gaps      |
| 40 – 59   | Candidato tem base mas falta experiencia relevante       |
| 0 – 39    | Pouca compatibilidade                                    |

## Regras de Negocio

- O usuario precisa ter a **OpenAI API Key** configurada em `PUT /api/settings`.
- O perfil do candidato enviado a IA inclui: cargo desejado, resumo profissional, skills, experiencias e projetos.
- Se o usuario nao tiver skills cadastradas, as primeiras 15 vagas sao selecionadas sem filtragem.
- Se houver skills, as vagas sao pre-filtradas por correspondencia textual com titulo, descricao e requisitos.
- Matches anteriores da mesma vaga para o mesmo usuario sao deletados antes de salvar o novo resultado.
- Resultados sao retornados ordenados por score (maior para menor).
- O resultado e armazenado em cache por 10 minutos por usuario.
- O cache e invalidado automaticamente apos extracao de curriculo.

## Fluxo Resumido

1. Extrai o `userId` do JWT no header `Authorization`.
2. Busca a OpenAI API Key do usuario nas settings — lanca `400` se ausente.
3. Busca todas as vagas no banco — lanca `400` se nao houver nenhuma.
4. Busca as skills do usuario.
5. Pre-filtra ate 15 vagas com maior correspondencia de skills.
6. Monta o texto de perfil do usuario (cargo, resumo, skills, experiencias, projetos).
7. Para cada vaga filtrada, envia perfil + vaga ao GPT e salva o resultado.
8. Retorna a lista de matches ordenada por score dentro do envelope `ApiResponse` com `200 OK`.
