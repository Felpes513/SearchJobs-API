## Descricao

Endpoints para consultar e atualizar os projetos do usuario autenticado. A operacao de atualizacao substitui completamente a lista existente.

## Autenticacao

- Necessita autenticacao: Sim
- Tipo: Bearer JWT
- Header: `Authorization: Bearer <access_token>`

---

## GET /api/profile/projects

Retorna todos os projetos do usuario autenticado.

### Request

#### Metodo e URL

`GET /api/profile/projects`

#### Headers

| Header        | Valor                   |
|---------------|-------------------------|
| Authorization | `Bearer <access_token>` |

#### Query Params

Nenhum.

#### Body

Nenhum.

### Response

#### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Projetos obtidos com sucesso",
  "data": [
    {
      "id": 1,
      "nome": "SearchJobs API",
      "descricao": "API REST para gerenciamento de vagas e perfis profissionais.",
      "stack": "Java, Spring Boot, PostgreSQL",
      "link": "https://github.com/usuario/searchjobs-api"
    }
  ]
}
```

#### Erros

`401 Unauthorized`

```json
{
  "success": false,
  "message": "Token ausente ou invalido"
}
```

## PUT /api/profile/projects

Substitui completamente os projetos do usuario autenticado pela lista fornecida.

### Request

#### Metodo e URL

`PUT /api/profile/projects`

#### Headers

| Header        | Valor                   |
|---------------|-------------------------|
| Authorization | `Bearer <access_token>` |
| Content-Type  | `application/json`      |

#### Body

```json
{
  "projetos": [
    {
      "nome": "SearchJobs API",
      "descricao": "API REST para gerenciamento de vagas e perfis profissionais.",
      "stack": "Java, Spring Boot, PostgreSQL",
      "link": "https://github.com/usuario/searchjobs-api"
    }
  ]
}
```

| Campo     | Tipo   | Obrigatorio | Descricao                             |
|-----------|--------|-------------|---------------------------------------|
| nome      | String | Nao         | Nome do projeto                       |
| descricao | String | Nao         | Descricao do projeto                  |
| stack     | String | Nao         | Tecnologias utilizadas                |
| link      | String | Nao         | URL do repositorio ou pagina do projeto |

### Response

#### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Projetos atualizados com sucesso",
  "data": [
    {
      "id": 2,
      "nome": "SearchJobs API",
      "descricao": "API REST para gerenciamento de vagas e perfis profissionais.",
      "stack": "Java, Spring Boot, PostgreSQL",
      "link": "https://github.com/usuario/searchjobs-api"
    }
  ]
}
```

#### Erros

`401 Unauthorized`

```json
{
  "success": false,
  "message": "Token ausente ou invalido"
}
```

## POST /api/profile/github/sync

Sincroniza os projetos do usuario autenticado com seus repositorios publicos no GitHub. O GitHub URL deve estar configurado no perfil antes de usar este endpoint.

### Request

#### Metodo e URL

`POST /api/profile/github/sync`

#### Headers

| Header        | Valor                   |
|---------------|-------------------------|
| Authorization | `Bearer <access_token>` |

#### Query Params

Nenhum.

#### Body

Nenhum.

### Response

#### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Projetos sincronizados com o GitHub com sucesso",
  "data": [
    {
      "id": 3,
      "nome": "searchjobs-api",
      "descricao": "API REST para gerenciamento de vagas e perfis profissionais.",
      "stack": "Java, Kotlin",
      "link": "https://github.com/usuario/searchjobs-api"
    }
  ]
}
```

#### Erros

`400 Bad Request`

```json
{
  "success": false,
  "message": "Configure seu GitHub no perfil antes de sincronizar"
}
```

`401 Unauthorized`

```json
{
  "success": false,
  "message": "Token ausente ou invalido"
}
```

## Regras de Negocio

- O ID do usuario e extraido diretamente do JWT.
- O PUT deleta todos os projetos existentes do usuario antes de salvar os novos.
- Uma lista vazia no body resultara na remocao de todos os projetos.
- A sincronizacao com GitHub (POST /github/sync) exige que o campo `githubUrl` esteja preenchido no perfil do usuario.
- Apenas repositorios publicos e que nao sejam forks sao importados (ate 30, ordenados por ultima atualizacao).
- A sincronizacao substitui completamente os projetos existentes do usuario.
- As linguagens de cada repositorio sao buscadas via API do GitHub e salvas no campo `stack`.

## Fluxo Resumido (PUT)

1. Extrai o `userId` do JWT no header `Authorization`.
2. Deleta todos os projetos existentes do usuario.
3. Persiste os novos projetos.
4. Retorna a lista atualizada dentro do envelope `ApiResponse` com `200 OK`.

## Fluxo Resumido (POST /github/sync)

1. Extrai o `userId` do JWT no header `Authorization`.
2. Busca o `githubUrl` do perfil do usuario.
3. Extrai o username a partir da URL do GitHub.
4. Consulta a API do GitHub: `GET /users/{username}/repos?per_page=30&sort=updated`.
5. Filtra repositorios publicos e que nao sejam forks.
6. Para cada repositorio, consulta as linguagens via `GET /repos/{username}/{repo}/languages`.
7. Deleta todos os projetos existentes do usuario.
8. Persiste os projetos importados.
9. Retorna a lista atualizada dentro do envelope `ApiResponse` com `200 OK`.
