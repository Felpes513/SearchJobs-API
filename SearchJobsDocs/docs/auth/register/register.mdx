## Descricao

Cria um novo usuario na plataforma a partir de nome, email e senha. A senha enviada no request nunca e persistida em texto puro: o servico aplica hash BCrypt antes de salvar.

## Autenticacao

- Necessita autenticacao: Nao
- Tipo: Publica

## Request

### Metodo e URL

`POST /api/auth/register`

### Body

```json
{
  "nome": "Felipe",
  "email": "felipe@email.com",
  "senha": "senha123"
}
```

### Query Params

Nenhum.

### Path Params

Nenhum.

## Response

### Sucesso

Status: `201 Created`

```json
{
  "mensagem": "Usuario cadastrado com sucesso"
}
```

### Erros

`409 Conflict`

Retornado quando o email informado ja esta cadastrado.

```json
{
  "status": 409,
  "erro": "Conflict",
  "mensagem": "Email ja cadastrado: felipe@email.com",
  "path": "/api/auth/register",
  "timestamp": "2024-01-15T10:30:00"
}
```

`422 Unprocessable Entity`

Retornado quando ha campos invalidos no body. O campo `campos` indica quais fields falharam.

```json
{
  "status": 422,
  "erro": "Validation Error",
  "mensagem": "Erro de validacao nos campos",
  "path": "/api/auth/register",
  "timestamp": "2024-01-15T10:30:00",
  "campos": {
    "nome": "Nome e obrigatorio",
    "senha": "Senha deve ter no minimo 6 caracteres"
  }
}
```

Motivos possiveis:

- `nome` ausente ou em branco
- `email` ausente ou em formato invalido
- `senha` ausente ou com menos de 6 caracteres

## Validacoes

| Campo  | Regra                              |
|--------|------------------------------------|
| nome   | Obrigatorio, nao pode ser em branco|
| email  | Obrigatorio, formato valido        |
| senha  | Obrigatoria, minimo 6 caracteres   |

## Regras de Negocio

- Nao permite cadastro com email duplicado (retorna `409`).
- Antes de persistir, a senha e convertida para hash com `BCryptPasswordEncoder`.
- O registro persiste um novo usuario na tabela `users`.
- O campo `createdAt` e definido automaticamente no `@PrePersist` da entidade JPA.

## Fluxo Resumido

1. Valida o body recebido com Bean Validation.
2. Verifica se o email ja existe no repositorio.
3. Gera o hash da senha com BCrypt.
4. Persiste o usuario.
5. Retorna mensagem de sucesso com `201 Created`.
