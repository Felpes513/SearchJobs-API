## Descricao

Autentica um usuario existente com email e senha. Quando as credenciais sao validas, a API retorna um `accessToken` e um `refreshToken` JWT.

## Autenticacao

- Necessita autenticacao: Nao
- Tipo: Publica

## Request

### Metodo e URL

`POST /api/auth/login`

### Body

```json
{
  "email": "felipe@email.com",
  "senha": "senha123"
}
```

### Query Params

Nenhum.

### Path Params

Nenhum.

## Response

### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Login realizado com sucesso",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

### Erros

`401 Unauthorized`

Retornado quando o email ou senha informados estao incorretos.

```json
{
  "success": false,
  "message": "Email ou senha invalidos"
}
```

`422 Unprocessable Entity`

Retornado quando ha campos invalidos no body.

```json
{
  "success": false,
  "message": "Erro de validacao nos campos",
  "errors": {
    "email": "Email invalido"
  }
}
```

Motivos possiveis:

- `email` ausente ou em formato invalido
- `senha` ausente

## Validacoes

| Campo | Regra                        |
|-------|------------------------------|
| email | Obrigatorio, formato valido  |
| senha | Obrigatoria                  |

## Regras de Negocio

- O login delega a validacao das credenciais ao `AuthenticationManager` do Spring Security.
- A busca do usuario para geracao dos tokens e feita pelo email.
- O `accessToken` e gerado com subject igual ao email do usuario e expira em 15 minutos.
- O `refreshToken` e gerado separadamente e expira em 7 dias.

## Fluxo Resumido

1. Valida o body recebido com Bean Validation.
2. Tenta autenticar email e senha no Spring Security.
3. Busca o usuario por email no repositorio.
4. Gera `accessToken` e `refreshToken`.
5. Retorna os tokens dentro do envelope `ApiResponse` com `200 OK`.
