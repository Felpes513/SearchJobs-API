---

## Descricao

Solicita o envio de um email com link de redefinicao de senha. Se o email informado estiver cadastrado, um token UUID e gerado, armazenado com validade de 30 minutos e enviado por email. Se o email nao existir, a requisicao retorna `200 OK` sem revelar essa informacao (comportamento silencioso por seguranca).

## Autenticacao

- Necessita autenticacao: Nao
- Tipo: Publica

## Request

### Metodo e URL

`POST /api/auth/forgot-password`

### Body

```json
{
  "email": "felipe@email.com"
}
```

### Query Params

Nenhum.

### Path Params

Nenhum.

## Response

### Sucesso

Status: `200 OK`

```json
{
  "success": true,
  "message": "Se o email estiver cadastrado, voce recebera um link de redefinicao de senha"
}
```

### Erros

`422 Unprocessable Entity`

Retornado quando ha campos invalidos no body.

```json
{
  "success": false,
  "message": "Erro de validacao nos campos",
  "errors": {
    "email": "Email invalido"
  }
}
```

Motivos possiveis:

- `email` ausente ou em formato invalido

## Validacoes

| Campo | Regra                       |
|-------|-----------------------------|
| email | Obrigatorio, formato valido |

## Regras de Negocio

- Se o email nao existir no banco, a operacao e ignorada silenciosamente — nenhum erro e retornado. Isso evita enumeracao de usuarios.
- Ao receber um email valido cadastrado, qualquer token anterior do mesmo usuario e removido antes de gerar o novo.
- O token gerado e um UUID aleatorio e expira em 30 minutos.
- O token e persistido na tabela `password_reset_tokens` com o campo `used = false`.
- O email e enviado com o token para que o usuario possa utiliza-lo na rota `POST /api/auth/reset-password`.

## Fluxo Resumido

1. Valida o body recebido com Bean Validation.
2. Busca o usuario pelo email no repositorio.
3. Se nao encontrado, retorna `200 OK` silenciosamente.
4. Remove tokens de redefinicao anteriores do usuario.
5. Gera um novo token UUID com validade de 30 minutos.
6. Persiste o token na tabela `password_reset_tokens`.
7. Envia o email com o token para o endereco informado.
8. Retorna `200 OK` com mensagem padrao.
