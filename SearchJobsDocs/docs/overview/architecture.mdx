## Descricao

Esta API segue arquitetura hexagonal, com responsabilidades distribuidas entre camadas de aplicacao, dominio e infraestrutura.

## Estrutura de Pastas

```
src/main/java/com/searchjobs/api/
├── application/
│   ├── dto/
│   │   ├── internal/         # DTOs internos (JSearchResponseDto)
│   │   ├── request/          # Contratos de entrada (LoginRequest, UpdateProfileRequest, ...)
│   │   └── response/         # Contratos de saida (AuthResponse, UserProfileResponse, JobResponse, ...)
│   └── service/              # Casos de uso aplicacionais (AuthService, UserProfileService, JobSearchService, ...)
├── config/                   # Configuracoes (SecurityConfig, SwaggerConfig, CorsConfig)
├── domain/
│   ├── exception/            # Excecoes de dominio (EmailAlreadyExistsException, ResumeNotFoundException, ...)
│   ├── model/                # Entidades de dominio (User, Resume, UserProfile, Job, UserSkill, ...)
│   └── port/
│       ├── in/               # Contratos de entrada dos casos de uso (AuthUseCase, UserProfileUseCase, JobSearchUseCase, ...)
│       └── out/              # Contratos de saida para persistencia e IA (UserRepository, JobRepository, JobSearchPort, ...)
└── infrastructure/
    ├── ai/                   # Integracao com OpenAI (OpenAiExtractionAdapter)
    ├── external/             # Integracao com APIs externas (JSearchAdapter)
    ├── persistence/
    │   ├── adapter/          # Implementacao dos ports de persistencia (UserProfileRepositoryAdapter, JobRepositoryAdapter, ...)
    │   ├── entity/           # Entidades JPA (UserJpaEntity, ResumeJpaEntity, UserProfileJpaEntity, JobJpaEntity, ...)
    │   └── repository/       # Spring Data JPA Repositories
    ├── security/             # JWT, filtro e integracao com Spring Security
    └── web/
        ├── controller/       # Controllers HTTP (AuthController, ResumeController, ProfileController, JobController)
        └── handler/          # Tratamento global de excecoes (GlobalExceptionHandler)
```

## Seguranca

- Rotas publicas: `/api/auth/**`, `/swagger-ui/**`, `/v3/api-docs/**`, `/swagger-ui.html`
- Demais rotas exigem autenticacao via JWT
- Politica de sessao: `STATELESS`
- Filtro principal: `JwtAuthFilter`
- Formato do header: `Authorization: Bearer <token>`

## Tratamento de Erros

Todos os erros seguem o padrao `ErrorResponse`:

```json
{
  "status": 422,
  "erro": "Validation Error",
  "mensagem": "Erro de validacao nos campos",
  "path": "/api/auth/register",
  "timestamp": "2024-01-15T10:30:00",
  "campos": {
    "email": "Email invalido",
    "senha": "Senha deve ter no minimo 6 caracteres"
  }
}
```

O campo `campos` so aparece em erros de validacao (`422`).

### Mapeamento de Excecoes

## Integracao com IA

O `OpenAiExtractionAdapter` e utilizado em dois contextos:

**Extracao de curriculo:** O servico `ResumeExtractionService` extrai texto do PDF usando Apache PDFBox e envia ao modelo GPT. O retorno e um JSON estruturado com os campos:

```json
{
  "nome": "...",
  "email": "...",
  "telefone": "...",
  "skills": [],
  "experiencias": [],
  "certificacoes": [],
  "projetos": []
}
```

**Rankeamento de vagas:** O servico `JobSearchService` utiliza o mesmo adaptador para ordenar as vagas retornadas pelo JSearch de acordo com a aderencia ao perfil do candidato. O modelo recebe as skills do usuario e as descricoes das vagas e retorna os IDs ordenados por relevancia.

## Integracao com JSearch

O `JSearchAdapter` implementa o port `JobSearchPort` e consome a API JSearch via RapidAPI. A busca e feita com base no `cargoDesejado` do perfil do usuario. Os parametros enviados sao:

| Parametro    | Valor                        |
|--------------|------------------------------|
| query        | Cargo desejado do usuario    |
| page         | 1                            |
| num_pages    | 1                            |
| country      | br                           |
| language     | pt                           |
| location     | Brasil                       |
| date_posted  | all                          |

As vagas retornadas sao persistidas na tabela `jobs` (upsert por `external_id`) e entao rankeadas pela IA antes de serem retornadas ao cliente.

## Observacoes

- O Swagger/OpenAPI declara esquema HTTP Bearer JWT global, disponivel em `/swagger-ui.html`.
- Arquivos PDF enviados sao armazenados localmente no diretorio configurado em `STORAGE_RESUMES_DIR`.
- Variaveis sensiveis (senha do banco, JWT secret, chave OpenAI) sao carregadas via `.env` com a biblioteca `spring-dotenv`.
